# 統合仕様書：Re:Spec（AI駆動開発支援プラットフォーム）

## 1. システム概要
本システム「Re:Spec」は、従来のウォーターフォール開発で作成されたドキュメントと、現代のAI駆動開発（AIDD）環境を接続するハブおよびプラットフォームです。古い仕様書をAIが解釈可能な構造化データへ変換し、AI駆動開発への橋渡しを行うとともに、仕様書が存在しないシステムに対してはリポジトリ解析による仕様書の自動生成を行うことで、開発プロセスの効率化と標準化を支援します。

## 2. 目的
本システムの目的は、古い仕様書の構造化データへの変換、および既存コードベースからの仕様書自動生成を通じて、AI駆動開発（Claude Code等）環境との連携を最適化し、開発プロセスの効率化と標準化を実現することです。

## 3. スコープ
本システムは、以下のプロセスを包含します。
* 古い仕様書（PDF等）の構造化データへの変換および品質レビュー
* GitHubリポジトリと連携した仕様書データの管理
* 既存コードベースからの仕様書自動生成
* AI駆動開発環境（Claude Code等）との連携基盤の提供

## 4. 用語定義
* **AI駆動開発（AIDD）**: AI（Claude Code等）を活用した開発手法。
* **構造化データ**: AIが読み取り・解釈可能な形式に変換された仕様書データ。
* **Re:Spec**: 本プラットフォームの名称。

## 6. 主要機能一覧
* **仕様書の自動変換**: 古い仕様書をAI可読な構造化データへ変換。

## 8. 価値の流れ
1. **入力**: 古い仕様書（ウォーターフォール資産）をシステムへ入力。
2. **内部処理**: 
    * 構造化データへの自動変換。
    * AIによる仕様書の漏れ指摘および修正。
    * 修正済み仕様書のGitHubへのプッシュ。
    * リポジトリコードからの仕様書自動作成。
3. **出力**: AI駆動開発（Claude Code等）での活用、または人間が読みやすい形式（Word等）への変換による社内共有。
