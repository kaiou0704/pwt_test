# 統合仕様書：Re:Spec（AI駆動開発支援プラットフォーム）

## 1. システム概要
本システム「Re:Spec」は、AI駆動開発（AIDD）時代における仕様書管理プラットフォームです。従来のウォーターフォール開発で作成されたドキュメントと、現代のAI駆動開発環境を構造化された仕様書（スペック）で接続し、開発プロセス全体を統合・効率化することを目的とします。

## 2. 目的
本システムの目的は、古い仕様書の構造化データへの変換、および既存コードベースからの仕様書自動生成を通じて、AI駆動開発（Claude Code等）環境との連携を最適化し、開発プロセスの効率化と標準化を実現することです。また、Notion PRD、Word/PDF、GitHubリポジトリを単一のプラットフォームで管理し、同期させる役割を担います。

## 3. スコープ
本プラットフォームは、Notion PM、Word PM、開発者、QAなど、開発に関わる全てのステークホルダーが同一環境で仕様書を管理することを想定しています。初期段階ではSIer、テストBPO、SaaS企業をターゲットとし、Notion連携をフックとした市場展開を行います。
* 古い仕様書（PDF/Word等）の構造化データへの変換および品質レビュー
* GitHubリポジトリと連携した仕様書データの管理
* 既存コードベースからの仕様書自動生成
* AI駆動開発環境（Claude Code等）との連携基盤の提供

## 4. 用語定義
* **AI駆動開発（AIDD）**: AI（Claude Code等）を活用した開発手法。
* **構造化データ**: AIが読み取り、精査・生成を行うために最適化された仕様書データ形式。
* **Re:Spec**: 本プラットフォームの名称。
* **FDE (Forward Deployed Engineer)**: 導入支援を行うエンジニア。SaaSの導入および大企業市場の開拓を担う。

### 5.4 主要な処理フロー
1. **変換**: Notion PRD、Word、コードを構造化データへ変換。
2. **検査**: 8観点AIインスペクションおよび品質スコア算出による仕様書の精査・修正。
3. **同期**: コード変更を検知し、仕様書を自動更新。
4. **通知**: Notion、Slack、Teams等へ乖離アラートを送信。
5. **出力**: Word/PDF、SPEC_template、監査証跡JSONの生成、およびNotionステータスの更新。
