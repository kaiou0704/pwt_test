# 外部インターフェース仕様書

## 1. 概要
本プラットフォームは、開発ツールおよびプロジェクト管理ツール（PMツール）と連携し、仕様書の変換、品質検査、およびコードとの同期を行うための外部インターフェースを提供する。Adapter Patternを採用し、特定のツールに依存しない中立的なインフラとして構成する。

## 2. 連携対象システム一覧
本プラットフォームは、以下の外部システムとの連携をサポートする。

### 2.1 開発・ソース管理ツール
* **GitHub**: 対応済み（WebhookおよびAPI連携）
* **Azure DevOps**: 計画中
* **GitLab / Bitbucket**: 将来的な対応を検討

### 2.2 プロジェクト管理・ドキュメントツール
* **Notion**: 対応済み（API連携）
* **Word / PDF**: 対応済み（ファイルインポート）
* **Confluence**: 計画中
* **Jira / Backlog**: 計画中

## 3. インターフェース機能詳細

### 3.1 AI Converter（ドキュメント変換エンジン）
外部ドキュメントを構造化Markdownへ変換、またはコードから仕様書を逆生成する。

* **入力形式**:
    * Word (.docx): テーブル、見出し、箇条書き構造の保持
    * PDF: OCR解析による構造抽出
    * Excel (.xlsx): マトリクス形式の要件表をMarkdownテーブルへ変換
    * Notion PRD: Notion API経由での取得および構造化
* **コード逆生成**:
    * リポジトリ解析によるファイル構造、依存関係、エントリポイントの特定
    * コード意図推論による機能仕様・IF仕様・制約のテンプレート生成

### 3.2 Sync & Integration Engine（同期・連携エンジン）
ツール間のデータ同期および変更検知を行う。

#### 3.2.1 GitHub連携（コード同期）
1. **検知**: PR/マージ時のWebhookによる変更検知
2. **解析**: AST（抽象構文木）およびセマンティクスレベルでの仕様書との差分比較
3. **分類**: 変更内容の重大度（重大/軽微/意図的）の自動分類
4. **実行**: 仕様書更新用PRの自動作成

#### 3.2.2 Notion連携（PRD同期）
1. **取込**: Notion APIによるPRDの取得と構造化Markdownへの変換
2. **検査**: AI Inspectorによる品質スコアリング結果のNotionコメントへの返却
3. **同期**: コード変更に伴うNotion PRDへの「要更新」通知およびステータス更新
4. **変換**: PRDからAIエージェントが消費可能なSPEC_templateへの自動変換

## 4. 技術的制約と方針
* **プラットフォーム中立性**: Gitホスティング機能自体は自前で構築せず、GitHub等の既存エコシステムをAPI/Webhook経由で活用する。
* **実装基盤**: 現在、GitHub連携はOctokitを使用して実装済み。
* **拡張性**: 将来的にGitLabやBitbucket等の対応を追加することで、特定のGitホスティングサービスへの依存を排除する設計とする。
